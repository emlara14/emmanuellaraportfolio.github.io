Project 4: Three-Tier Distributed Web-Based Application

I developed a distributed three-tier web application using Java servlets and JSP technology running on a Tomcat container to maintain a persistent MySQL database[cite: 4, 164].

  • Multi-Level Authentication: An HTML landing page facilitates user authentication via a servlet that validates credentials against a dedicated MySQL credentialsDB[cite: 14, 15, 17]. Users are redirected to specific interfaces for root, client, or accountant roles[cite: 20, 21, 24].

  • Logic & SQL Interface: Second-tier servlets handle arbitrary SQL processing for root and client users[cite: 37, 43]. I implemented server-side business logic to automatically increment supplier status by 5 when shipment quantities reach 100 or more, specifically without using database triggers[cite: 39, 41, 65].

  • Data & Reports: The backend MySQL DBMS enforces referential integrity for shipment records[cite: 10, 12, 49]. Accountant-level reports are generated by executing remote stored procedures via the CallableStatement interface[cite: 30, 47, 48].

  • Secure Access: Database connections are managed through properties files to ensure distinct credential-based access for root, client, and accountant privilege levels[cite: 32, 33, 34].
